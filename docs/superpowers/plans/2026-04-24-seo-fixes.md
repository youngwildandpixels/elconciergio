## Task 3 : Prerender SPA (vite-plugin-prerender)

**Problème** : Le site est une React SPA rendue 100% côté client. Googlebot voit `<div id="root"></div>` vide si le JS ne s'exécute pas. Le prerender génère des snapshots HTML statiques au moment du build — aucun changement de code applicatif requis.

**Files:**
- Modify: `package.json` (dépendance dev)
- Modify: `vite.config.ts` (plugin prerender)

- [ ] **Étape 1 : Installer `vite-plugin-prerender`**

```bash
npm install --save-dev vite-plugin-prerender
```

Vérifier que `package.json` contient `"vite-plugin-prerender"` dans `devDependencies`.

- [ ] **Étape 2 : Lire `vite.config.ts` avant de le modifier**

Ouvrir [vite.config.ts](vite.config.ts) et noter les plugins déjà présents pour ne pas les écraser.

- [ ] **Étape 3 : Ajouter le plugin dans `vite.config.ts`**

Ajouter en haut des imports :
```ts
import prerender from 'vite-plugin-prerender';
import path from 'node:path';
```

Dans le tableau `plugins`, ajouter après les plugins existants :
```ts
prerender({
  staticDir: path.join(__dirname, 'dist'),
  routes: ['/'],
}),
```

> Si la config Hostinger utilise une `base` (ex: `/elconciergio/`), remplacer `routes: ['/']` par `routes: ['/elconciergio/']`.

- [ ] **Étape 4 : Builder et vérifier le HTML généré**

```bash
npm run build
```

Vérifier que `dist/index.html` (ou `dist/elconciergio/index.html`) contient maintenant du texte visible comme "La conciergerie WhatsApp" dans le HTML brut :
```bash
grep "conciergerie WhatsApp" dist/index.html
```
Résultat attendu : la phrase apparaît dans le HTML statique.

- [ ] **Étape 5 : Tester le build localement**

```bash
npm run preview
```

Ouvrir `http://localhost:4173` → le site doit s'afficher normalement.

- [ ] **Étape 6 : Commit**

```bash
git add vite.config.ts package.json package-lock.json
git commit -m "feat(seo): add vite-plugin-prerender for SSR-equivalent static HTML snapshots"
```
